# GraphRAG 中央苏区史问答系统 — 使用说明

## 项目架构总览

```
E:\graphRAG\output\
├── backend\          ← 问答系统后端（FastAPI，端口 8084）
├── vue\              ← 后台管理前端（Vue3，端口 5173）
├── vue_backend\      ← 后台管理后端（FastAPI，端口 8000）
├── frontnew\         ← 问答系统旧版前端（Flask，端口 5000，可选）
├── vue2\             ← 后台管理前端备份（与 vue\ 一致）
└── *.parquet         ← GraphRAG 数据文件（原始知识图谱数据）
```

**依赖服务：**
- MySQL（端口 3306，数据库 `graphrag_admin2`）
- Neo4j（端口 7687）

## 一、首次配置

### 1. 配置 `backend/.env`

在 `E:\graphRAG\output\backend\` 目录下创建 `.env` 文件（如已有则编辑）：

```env
# LLM 配置（选一个填写）
LLM_PROVIDER=dashscope
DASHSCOPE_API_KEY=你的通义千问API Key
DASHSCOPE_MODEL=qwen-plus

# 其他 LLM 选项（二选一填写即可）
# LLM_PROVIDER=deepseek
# DEEPSEEK_API_KEY=你的DeepSeek API Key

# MySQL 数据库
DATABASE_URL=mysql+pymysql://root:root@localhost:3306/graphrag_admin2

# Neo4j 图数据库
NEO4J_URI=neo4j://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=12345678

# GraphRAG 数据路径（..表示指向 output 目录，即 parquet 文件所在位置）
GRAPHRAG_DATA_PATH=..
```

### 2. 配置 `vue_backend/.env`

在 `E:\graphRAG\output\vue_backend\` 目录下创建 `.env` 文件：

```env
# 后台管理系统数据库（与 backend 共用）
DATABASE_URL=mysql+pymysql://root:root@localhost:3306/graphrag_admin2

# 问答系统地址（vue_backend 从这里同步数据）
QA_SYSTEM_URL=http://localhost:8084
```

### 3. 创建 MySQL 数据库

在 MySQL 中执行：
```sql
CREATE DATABASE IF NOT EXISTS graphrag_admin2 CHARACTER SET utf8mb4;
```

### 4. 安装 Python 依赖

打开两个终端，分别执行：

```bash
# 问答后端依赖
cd E:\graphRAG\output\backend
pip install -r requirements.txt

# 后台管理后端依赖
cd E:\graphRAG\output\vue_backend
pip install -r requirements.txt
```

### 5. 安装前端依赖

```bash
cd E:\graphRAG\output\vue
npm install
```

## 二、每次启动顺序

**必须按顺序启动，共需开 3 个终端窗口：**

### 终端 1 — 问答系统后端（端口 8084）

```bash
cd E:\graphRAG\output\backend
python main.py
```

启动成功标志：
```
✅ 环境变量已加载
✅ Neo4j 连接成功
✅ GraphRAG 数据加载完成！
🚀 启动服务器: http://0.0.0.0:8084
```

### 终端 2 — 后台管理后端（端口 8000）

```bash
cd E:\graphRAG\output\vue_backend
python app.py
```

启动成功标志：
```
✅ 数据库表已初始化
GraphRAG 后台管理系统 API 启动
API 文档: http://localhost:8000/api/docs
```

### 终端 3 — 后台管理前端（端口 5173）

```bash
cd E:\graphRAG\output\vue
npm run dev
```

启动成功标志：
```
  ➜  Local:   http://localhost:5173/
```

## 四、旧版问答前端（frontnew，推荐使用）

`frontnew/index_updated.html` 是独立的聊天界面，直接与问答后端（端口 8084）通信。

**使用方式（二选一）：**

**方式一：直接双击打开（最简单）**
直接用浏览器打开 `E:\graphRAG\output\frontnew\index_updated.html`

**方式二：通过 Flask 启动（如果方式一有跨域问题）**
```bash
cd E:\graphRAG\output\frontnew
pip install flask requests
python app_new.py
# 访问 http://localhost:5000
```

> 注意：方式一需要 `backend/main.py` 已启动（端口 8084）。CORS 已配置为允许所有来源。

## 五、常见问题

### Neo4j 连接失败
问答后端会打印警告但不会崩溃，问答功能正常，只是**图谱可视化不可用**。确认 Neo4j Desktop 已启动，或修改 `.env` 中的密码。

### `graphrag_admin2` 数据库不存在
后端启动时会自动创建数据表，但**数据库本身需要手动创建**（见上方 SQL 命令）。

### 端口被占用
查找并结束占用进程：
```bash
# Windows
netstat -ano | findstr :8084
taskkill /PID <进程号> /F
```

### LLM 未配置 / 回答为空
检查 `backend/.env` 中的 `LLM_PROVIDER` 和对应的 `API_KEY` 是否填写正确。后端启动时会打印：
- `✅ LLM 已配置: dashscope - qwen-plus` — 正常
- `⚠️  LLM API Key 未配置` — 需检查 `.env`

### 切换 LLM 模型
只需修改 `backend/.env`，然后重启终端 1 即可。支持的提供商：`dashscope`、`deepseek`、`groq`、`zhipu`、`siliconflow`。详细说明见 `E:\graphRAG\output\LLM_CHANGE_GUIDE.md`。

## 六、数据文件说明

`E:\graphRAG\output\` 根目录下的 `.parquet` 文件是 GraphRAG 生成的知识图谱数据，后端启动时自动加载，**不需要手动操作**：

| 文件 | 内容 |
|------|------|
| `entities.parquet` | 实体（人物、地点、事件等） |
| `relationships.parquet` | 实体间关系 |
| `communities.parquet` | 社区分组 |
| `community_reports.parquet` | 社区摘要报告（问答的核心数据） |
| `text_units.parquet` | 原始文本单元 |
| `documents.parquet` | 原始文档 |
